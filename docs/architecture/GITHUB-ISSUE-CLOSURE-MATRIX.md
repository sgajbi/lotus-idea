2. A row can be `locally_fixed` only when implementation, meaningful tests,
   same-pattern scan, docs/wiki/context decision, and PR close intent are
   present on the branch.
3. A row can be `merged_main` only after it records Main Releasability and
   CodeQL evidence on the merged commit, wiki publication, issue closure, and
   local and remote branch cleanup.

   the issue only after merge.

   `make lint`.
6. `MERGED_MAIN_ISSUES` in the gate freezes completed issue state so a later
   documentation edit cannot silently regress merged truth to branch-local
   wording. Closure reconciliation must also remove active-work GitHub labels
   such as `status/pr-open`, `status/fixed-local`, and `status/in-progress`.
