# ADR-0002: Scaffold And Repository Foundation

Status: Accepted

Date: 2026-06-21

## Context

New Lotus services should be created from platform automation so they inherit
the same CI lane model, quality gates, repository context, source wiki posture,
supported-feature registry, Docker baseline, and layered backend structure.

The user requested repository creation, public GitHub setup, protected `main`,
ADRs, and RFCs before business implementation.

## Decision

Scaffold `lotus-idea` with `lotus-platform/automation/New-Lotus-Service.ps1`
using the `domain-service` profile and `python-fastapi` runtime.

Initial repository work is limited to foundation truth:

1. README and repository engineering context,
2. ADRs,
3. RFC suite,
4. source wiki,
5. supported-feature posture,
6. platform registry updates generated by the scaffold.

## Consequences

The repository starts in an enterprise-ready direction without pretending that
business features are complete. Future feature work must move through RFCs and
must update implementation, tests, API docs, supported features, endpoint
certification, wiki, and evidence together.
